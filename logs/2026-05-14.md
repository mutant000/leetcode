## 13. Roman to Integer

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Hash Table, Math, String
- Status: Accepted, 3999 / 3999 testcases passed
- Submission: https://leetcode.com/problems/roman-to-integer/submissions/2002614897/
- Runtime: 9 ms
- Memory: 19.25 MB

## 思路

用哈希表保存七个罗马字符的数值，从左到右扫描。

如果当前字符比右边字符小，就把当前值减掉；否则加上当前值。这样可以统一处理 `IV`、`IX`、`XL`、`XC`、`CD`、`CM` 这些减法组合。

## 136. Single Number

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Array, Bit Manipulation
- Status: Accepted, 61 / 61 testcases passed
- Submission: https://leetcode.com/problems/single-number/submissions/2002650401/
- Runtime: 0 ms
- Memory: 21.08 MB

## 思路

利用异或抵消成对出现的数字：`x ^ x = 0`，`x ^ 0 = x`，并且异或满足交换律和结合律。

把数组中所有数字依次异或，重复两次的数字最终都会归零，只剩下出现一次的数字。

## 160. Intersection of Two Linked Lists

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Linked List, Two Pointers
- Status: Accepted, 41 / 41 testcases passed
- Submission: https://leetcode.com/problems/intersection-of-two-linked-lists/submissions/2002602285/
- Runtime: 116 ms
- Memory: 38.02 MB

## 思路

用两个指针分别从两条链表头出发。指针走到末尾后切换到另一条链表头，这样两个指针最终都会走过 `m + n` 的总长度。

如果链表相交，它们会在交点相遇；如果不相交，它们会同时到达 `None`。

## 206. Reverse Linked List

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Linked List, Recursion
- Status: Accepted, 28 / 28 testcases passed
- Submission: https://leetcode.com/problems/reverse-linked-list/submissions/2002629856/
- Runtime: 0 ms
- Memory: 20.33 MB

## 思路

用 `prev` 和 `current` 原地反转指针。

每一步先保存 `current.next`，再把当前节点指向已经反转好的前半段，然后同步推进两个指针。遍历结束后，`prev` 就是新的链表头。

## 234. Palindrome Linked List

- Status: Accepted, 93 / 93 testcases passed
- Submission: https://leetcode.com/problems/palindrome-linked-list/submissions/2002643490/
- Runtime: 38 ms
- Memory: 53.55 MB

## 思路

用快慢指针找到链表后半段起点，再原地反转后半段。

从原头节点和反转后的后半段头节点同步比较；如果遇到不相等就标记失败。最后再把后半段反转回来，避免提交之外的测试复用链表时被修改。

## 283. Move Zeroes

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Array, Two Pointers
- Status: Accepted, 75 / 75 testcases passed
- Submission: https://leetcode.com/problems/move-zeroes/submissions/2002668407/
- Runtime: 0 ms
- Memory: 20.21 MB

## 思路

用 `write` 指针维护下一个非零元素应该写入的位置。

先按原顺序把所有非零元素写到数组前部，再把剩余位置补成 `0`。这样不会创建额外数组，并且能保持非零元素的相对顺序。

## 169. Majority Element

- Track: LeetCode Top 100 Liked
- Difficulty: Easy
- Topic: Array, Hash Table, Divide and Conquer, Sorting, Counting
- Status: Accepted, 53 / 53 testcases passed
- Submission: https://leetcode.com/problems/majority-element/submissions/2002801813/
- Runtime: 6 ms
- Memory: 21.11 MB

## 思路

用 Boyer-Moore 投票法维护候选数和计数。

当计数为 `0` 时，当前数字成为新的候选数；遇到候选数就加一，遇到其他数字就减一。因为多数元素出现次数超过一半，它和其他元素抵消后仍会留下，最终候选数就是答案。
