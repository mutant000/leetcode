# 39. Combination Sum

## 思路

用回溯枚举组合，并让路径始终按候选数的排序顺序前进。

先把 `candidates` 排序。递归函数维护两个状态：

1. `start` 表示本层可以从哪个下标开始选择，避免同一组数字因为排列顺序不同被重复加入。
2. `remaining` 表示距离 `target` 还差多少。

选择 `candidates[index]` 后，下一层仍然从 `index` 开始，因为同一个数字可以重复使用。若当前候选数已经大于 `remaining`，后面的候选数也都会更大，可以直接剪枝。

## 复杂度

- Time: `O(N^(T/M))` 量级，其中 `N` 是候选数个数，`T` 是 `target`，`M` 是最小候选数；实际会被排序剪枝和答案数量限制压缩。
- Space: `O(T/M)` 递归路径深度；结果列表额外保存所有合法组合。

## 边界

- `target` 小于所有候选数时返回空列表。
- 候选数可以重复使用，所以递归时传入当前下标而不是 `index + 1`。
- 输入候选数互不相同，但原始顺序不保证有序，需要排序后才能安全剪枝。
