# 432. All O-one Data Structure

## 思路

使用“哈希表 + 双向桶链表”：

- 每个桶保存一个计数值，以及拥有这个计数的所有键。
- 桶按照计数从小到大连接。
- `key_to_bucket` 记录每个键当前位于哪个桶。

`inc` 和 `dec` 只会把键移动到相邻计数的桶。如果目标桶不存在，就在当前位置旁边创建；原桶变空后立即删除。

最小键位于头哨兵后的第一个桶，最大键位于尾哨兵前的最后一个桶。集合可以在常数时间返回任意一个键。

## 复杂度

- `inc`、`dec`、`getMaxKey`、`getMinKey`：平均时间复杂度均为 `O(1)`
- 空间复杂度：`O(n)`，`n` 是不同键的数量
