# 440. K-th Smallest in Lexicographical Order

## 思路

把 `1` 到 `n` 看成一棵按数字前缀组织的十叉树。例如前缀 `1` 的子树包含 `1、10、100、101...`。

`count_steps(first, next_prefix)` 计算两个相邻前缀之间一共有多少个合法数字：

- 如果这棵前缀子树的节点数不超过剩余的 `k`，就一次跳过整棵子树，移动到下一个兄弟前缀。
- 否则答案在当前子树中，进入 `current * 10`。

这样不需要真正生成或排序 `1` 到 `n`。

## 复杂度

- 时间复杂度：`O(log^2 n)`
- 空间复杂度：`O(1)`
