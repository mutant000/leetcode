# 79. Word Search

## 思路

先做两层剪枝，再回溯搜索：

1. 统计整个棋盘字符频次。如果 `word` 某个字符需要次数超过棋盘提供次数，直接返回 `False`。
2. 如果 `word` 首字符在棋盘里出现次数比尾字符更多，就把 `word` 反转。这样可以尽量从更稀有的字符开始搜，减少起点数量。

之后对每个可能起点做 DFS：

- 当前格子字符必须匹配 `word[index]`。
- 匹配后先把当前格子临时标记成 `#`，避免同一路径重复使用。
- 只向上下左右四个方向继续搜索下一个字符。
- 回溯时再把格子恢复原值。

## 复杂度

- 时间复杂度：`O(m * n * 3^(L - 1))`，`L` 是 `word` 长度。首步之后每层最多继续走 3 个方向。
- 空间复杂度：`O(L)`，主要是递归栈深度。

## 边界情况

- `word` 中存在棋盘里根本不够用的字符。
- 搜索路径需要拐弯，不能重复使用同一个格子。
- 只有一个格子，或者 `word` 长度为 1。
