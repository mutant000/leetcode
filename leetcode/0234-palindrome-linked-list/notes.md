# 234. Palindrome Linked List

## 思路

用快慢指针找到链表后半段起点，再原地反转后半段。

1. `slow` 每次走一步，`fast` 每次走两步；循环结束时，`slow` 位于中点或后半段开头。
2. 反转从 `slow` 开始的后半段。
3. 从原头节点和反转后的后半段头节点同步比较。
4. 比较结束后再把后半段反转回来，保持输入链表结构不变。

奇数长度时，中间节点会被放进后半段。它最后才被比较，不影响回文判断。

## 复杂度

- Time: `O(n)`
- Space: `O(1)`

## 边界

- 单节点链表一定是回文。
- 偶数长度时两个半段长度相同。
- 奇数长度时中间节点无需特殊跳过。
- 发现不匹配后也要恢复后半段链表。
