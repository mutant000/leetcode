# 437. Path Sum III

## 思路

路径只能从父节点往子节点走，所以 DFS 时维护从根到当前节点的前缀和。

如果当前前缀和是 `current_sum`，想找到一段向下路径和为 `targetSum`，就需要之前某个祖先前缀和等于 `current_sum - targetSum`。用哈希表记录当前递归路径上的前缀和出现次数即可。

进入节点时加入当前前缀和，递归左右子树；离开节点时再撤销，保证哈希表只表示当前路径。

## 复杂度

- Time: `O(n)`
- Space: `O(h)`，递归栈和当前路径上的前缀和计数。

## 边界

- 空树返回 `0`。
- 节点值可能很大，不能用剪枝假设路径和单调。
- 多个相同前缀和需要计数，而不是只存是否出现。
