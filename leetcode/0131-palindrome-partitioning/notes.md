# 131. Palindrome Partitioning

## 思路

先用动态规划预处理任意子串是否为回文，再做回溯枚举所有切分方案。

定义 `is_palindrome[i][j]` 表示 `s[i:j+1]` 是否是回文串：

- 只有当 `s[i] == s[j]` 时才可能成立。
- 如果区间长度不超过 3，那么首尾相等就已经足够。
- 更长的区间则依赖内部子串 `is_palindrome[i + 1][j - 1]`。

这样回溯时就不需要每次再线性检查子串是否回文。DFS 从当前位置 `start` 出发，依次尝试所有满足回文条件的结尾 `end`，把当前子串加入 `path`，递归处理剩余后缀，再撤销选择。

## 复杂度

- Time: `O(n * 2^n)`，预处理回文表是 `O(n^2)`，回溯需要枚举所有合法切分，最坏情况下结果数量是指数级。
- Space: `O(n^2)`，主要来自回文判定表；递归栈和当前路径最多再占 `O(n)`。

## 边界

- `s.length = 1` 时，答案只有 `[[s]]`。
- 全部字符相同，例如 `"aaaa"`，会产生大量合法切分，需要正确枚举全部结果。
- 完全没有长度大于 1 的回文子串时，只能逐字符切分。
