# 46. Permutations

## 思路

用回溯逐位构造排列。

`path` 保存当前已经选好的前缀，`used[index]` 表示 `nums[index]` 是否已经在当前排列中。每一层枚举所有还没使用过的数字，放入路径后继续递归；递归返回时撤销选择，让同一位置可以尝试其他数字。

当 `path` 长度等于 `nums` 长度时，说明一个完整排列已经生成，把当前路径的拷贝加入答案。

## 复杂度

- Time: `O(n * n!)`，一共有 `n!` 个排列，每个排列复制需要 `O(n)`。
- Space: `O(n)`，递归深度、路径和 `used` 数组都是 `O(n)`；结果列表不计入额外空间。

## 边界

- `nums.length = 1` 时，唯一排列就是原数组本身。
- 输入数字互不相同，所以不需要去重。
- 数字可能为负数或 `0`，只影响值本身，不影响回溯选择逻辑。
